# Contributing

Thanks for your interest in Spire.

- Fork the repo, create a **feature branch**, and open a **pull request** against `main`.
- Use **Conventional Commits** (`feat:`, `fix:`, `chore:`, `docs:`, etc.) so the history stays readable.
- Run **`npm run tsc`** (or `tsc --noEmit` as configured in the project) before opening a PR.
- For **large changes**, open an issue first so we can align on direction.
- **No new npm dependencies** without discussion in an issue or PR — they affect install size, security, and maintenance.

If something is unclear, ask in the issue or PR. Small fixes and docs are always welcome.
